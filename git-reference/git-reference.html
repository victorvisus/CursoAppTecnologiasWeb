Guía de Referencia Git

Comandos esenciales para tu flujo de trabajo diario

Configurar Git globalmente

Para configurar Git globalmente, ejecuta los siguientes comandos en la terminal: 

git config --global user.name "Tu nombre"

Y luego:

git config --global user.email "TuCorreo@ejemplo.com"

Configurar Git en la carpeta de trabajo

Para configurar Git en la carpeta de trabajo, ejecuta los siguientes comandos en la terminal dentro de la carpeta: 

git config user.name "Tu nombre"

Y luego:

git config user.email "TuCorreo@ejemplo.com"

Comprobar las configuraciones

Para comprobar las configuraciones de Git tanto global como local, ejecuta los siguientes comandos en la terminal: 

git config --global --list

Y luego:

git config --local --list

Iniciar un repositorio en local

git init

Inicia un nuevo repositorio en el directorio actual

Crear un repositorio en GitHub

Crear un nuevo repositorio en GitHub

Ve a la página de GitHub, crea un nuevo repositorio vacío

Unificar el repositorio local y el remoto en GitHub

git remote add origin https://github.com/tu-usuario/tu-repositorio.git

Agrega el repositorio remoto creado en GitHub

git push -u origin master

Sube tu repositorio local a GitHub

El Ciclo Diario (Guardar cambios)

git status

Te dice qué archivos han cambiado y en qué rama estás

git add .

Prepara todos los cambios para el commit

git add <archivo>

Prepara solo un archivo específico

git commit -m "mensaje"

Crea una "foto" de tus cambios con una descripción

git push origin <rama>

Sube tus commits al repositorio de GitHub

git pull origin <rama>

Trae los cambios de GitHub a tu PC (útil si trabajas en varios PCs)

Gestión de Ramas (Ideal para tus módulos)

Crear y saltar a una rama nueva:

git checkout -b nombre-rama

Listar todas las ramas:

git branch

Cambiar de una rama a otra:

git checkout nombre-rama

Fusionar una rama en la que estás (Merge):

git merge rama-que-quieres-traer

Ej: estando en main, haces git merge lenguaje-de-marcas

Borrar una rama local:

git branch -d nombre-rama

Consultas y "Socorro"

Ver el historial de cambios:

git log --oneline

Muestra los commits de forma resumida

Deshacer cambios locales (sin commit):

git restore <archivo>

Vuelve al estado del último commit

Corregir el último mensaje de commit:

git commit --amend -m "Nuevo mensaje"

Truco para tu caso (Sincronización)

Como tienes el repo en una ruta de Windows específica, a veces los permisos o archivos ocultos dan guerra. Si alguna vez notas que el git status te muestra archivos que deberían estar ignorados (por el .gitignore), usa este "reseteo" del índice: 

git rm -r --cached .
git add .
git commit -m "Limpieza de archivos ignorados"
