HTML Semantico

Fundamentos de la Semántica Web

¿Qué es el HTML Semántico?

El HTML semántico es el uso de etiquetas que comunican claramente su significado tanto a los navegadores como a los desarrolladores. En lugar de usar etiquetas genéricas, utilizamos elementos que describen el contenido que albergan de forma precisa.

El valor de la accesibilidad

Utilizar etiquetas correctas permite que las tecnologías de asistencia comprendan la jerarquía y la importancia de cada parte de la página. Esto es vital para que personas con discapacidad visual naveguen de forma autónoma.

Beneficios para el SEO

Indexación inteligente

Los motores de búsqueda priorizan las páginas bien estructuradas. Al usar etiquetas como <article> o <section>, estamos indicando qué partes del contenido son relevantes, mejorando así el posicionamiento orgánico. ¡No lo ignores!

Estructura de Bloque y Organización

Principales etiquetas de organización

  • Seccionamiento: Elementos como <main> y <nav> que definen las grandes áreas de la interfaz.
  • Contenido independiente: El uso de <article> para bloques que tienen sentido completo por sí mismos.
  • Estructuras de agrupación comunes:
    • <header>: Contiene habitualmente títulos, logos o navegación.
    • <footer>: Información de copyright, enlaces legales o redes sociales.
    • <aside>: Contenido tangencial o barras laterales relacionadas de forma indirecta.

Comparativa: Estructura Tradicional vs. Semántica

¿Por qué dejar de usar solo divs?

Antiguamente, la estructura de una web dependía exclusivamente de etiquetas <div>. Hoy, el estándar moderno prioriza el significado.

Evolución de etiquetas genéricas a semánticas
Elemento Antiguo Elemento Moderno Propósito Principal
<div id="header"> <header> Cabecera de página o sección.
<div class="nav"> <nav> Contenedor de enlaces de navegación.
<div class="post"> <article> Contenido independiente (noticia, post).
Nota: Atributos como scope y headers mejoran la lectura para screen readers.
